OVERVIEW :::: This project is an implementation of a RAG system built using Google GenAI (Gemini) and ChromaDB, focused on understanding the full pipeline rather than just using pre-built tools. It loads a text document, splits it into meaningful chunks, converts them into embeddings, then stores them in a vector database. When the user asks a question, the system retrieves relevant chunks using a mix of semantic similarity and keyword-based search, ranks them with a combined scoring method, and sends the best context to the model to generate a grounded answer. It also includes a basic confidence threshold to filter weak matches and a logging system that tracks questions, answers, scores, and user feedback for analysis of the model. 
Right now it’s terminal-based and works with text files, but the goal is to extend it into a more scalable system with a UI, document uploads (like PDFs), and better retrieval and evaluation methods.
